# Contributing to 100tools

Thanks for thinking about contributing. 100tools is a small project with a focused scope, and the community is the single biggest reason we're able to expand it.

## TL;DR

The application source for 100tools is maintained in a **private repository** so we can iterate quickly without code-review overhead. This public repository is for:

- 📋 **Issue tracking** (bug reports + tool requests)
- 💬 **Discussions** (ideas, feedback, support)
- 📚 **Public docs** (this README, FAQ, roadmap, examples)
- 🌍 **Future translations** of tool copy

You can still meaningfully contribute without touching code.

---

## Ways to contribute

### ⭐ Star the repository
The single highest-leverage thing you can do. Stars are how GitHub surfaces this repo in search and discovery.

### 🐛 Report a bug
Open an issue using the **[Bug report template](../../issues/new?template=bug-report.md)**. Include:
- The tool's URL (e.g. `100tools.site/tools/json-formatter`)
- Browser + OS
- Steps to reproduce
- Expected vs actual behaviour
- A minimal sample input if possible

Reproducible bugs get fixed within days.

### 💡 Suggest a new tool
Open an issue using the **[Tool request template](../../issues/new?template=tool-request.md)**. Describe:
- What the tool does (one sentence)
- The category it belongs to
- A typical input and expected output
- Any reference tools you've used before

We prioritise by 👍 reactions on tool requests, so feel free to thumbs-up someone else's idea.

### ✍️ Improve the docs
Typos, broken links, unclear FAQ answers, missing use-case examples — all welcome via pull request. The repo accepts PRs against:

- `README.md` — main landing
- `FAQ.md` — frequently asked questions
- `ROADMAP.md` — keep dates and statuses fresh
- `docs/*.md` — category overviews and use-case writeups
- `examples/*.md` — real-world workflows that combine multiple tools

Open the PR against the `main` branch. We'll review within a few days.

### 🌍 Translation (planned)
Once localisation infrastructure is in place (see [Roadmap](ROADMAP.md)), native-speaker review of tool copy will be invaluable. Watch this repo to get notified.

### 💬 Help in discussions
[GitHub Discussions](../../discussions) is where people ask "is there a tool for X?" and "how do I do Y with 100tools?" Answering one of those is genuinely useful.

---

## What we can't accept

- **Code contributions to the tool implementations themselves.** The application source is private. If a tool has a bug, file a bug report — we'll fix it on our side.
- **Affiliate links** or promotional content in docs.
- **AI-generated boilerplate** issues or PRs. We can tell, and it wastes maintainer time.

---

## Code of conduct

Be kind. Assume good faith. We have low tolerance for personal attacks, harassment, or aggressive behaviour in issues, PRs, or discussions. Maintainers will moderate freely.

---

## Recognition

Contributors whose issues or PRs ship in a release will be credited in [CHANGELOG.md](CHANGELOG.md) and (if they want) linked from the website's credits page.
